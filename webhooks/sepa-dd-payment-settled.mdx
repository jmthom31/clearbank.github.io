---
title: "SEPA Direct Debits Payment Settled webhook"
version: 1
webhook: true
---

Sent when a SEPA Direct Debit payment has been settled. Follows the Incoming Collection Received webhook.

**Note:** You can correlate settled payments with the original collection using the `paymentId` and `transactionId` as these correspond to the same fields in the Incoming Collection Received webhook.

#### Request body

```json
{
  "Type": "Sepa.DirectDebits.Payment.Settled",
  "Version": 1,
  "Payload": {...},
  "Nonce":
}
```

#### Payload Fields

| Field                       | Type   | Format | Required | Min length | Max length | Description                                                                                                                                                                                                                          |
| --------------------------- | ------ | ------ | -------- | ---------- | ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `paymentId`                 | string | uuid   | Yes      | -          | -          | Unique ClearBank identifier for the payment. Previously advised in the Incoming Collection Received webhook. Use this to correlate the settled payment with the original collection.                                                 |
| `transactionId`             | string | uuid   | Yes      | -          | -          | Unique ClearBank identifier for the transaction applied to the advised Debtor Account. Previously advised in the Incoming Collection Received webhook.                                                                               |
| `settlementDate`            | string | date   | Yes      | -          | -          | Date on which the amount of money ceases to be available to the agent that owes it and becomes available to the agent to which it is due. Format: `YYYY-MM-DD`.                                                                      |
| `debitCreditCode`           | string | -      | Yes      | 5          | 6          | Advises if the payment is a debit or credit. `Debit` = standard Direct Debit collection (debit from debtor). `Credit` = credit payment (for a return or refund scenario).                                                            |
| `interbankSettlementAmount` | object | -      | Yes      | -          | -          | Settled amount in the interbank space. See `InstructedAmount`.                                                                                                                                                                       |
| `debtorAccount`             | object | -      | Yes      | -          | -          | Account of the debtor to which the debit entry was applied. See `PartyAccount`.                                                                                                                                                      |
| `paymentType`               | string | -      | No       | 1          | 70         | Advises if the settled transaction is for a return, a refund, or another business scenario. Possible values: `Return`, `Authorised Refund`, `Unauthorised Refund`, `Reversal`. Only populated for non-standard collection scenarios. |

##### InstructedAmount

| Field      | Type    | Required | Constraints       | Pattern      | Description                                                        |
| ---------- | ------- | -------- | ----------------- | ------------ | ------------------------------------------------------------------ |
| `amount`   | decimal | Yes      | 0.01â€“999999999.99 | -            | The settled amount. Must be 0.01 or more and 999999999.99 or less. |
| `currency` | string  | Yes      | Must be `EUR`     | `^[A-Z]{3}$` | Currency of the settled amount. Only EUR is allowed.               |

##### PartyAccount (DebtorAccount)

| Field  | Type   | Required | Min length | Max length | Pattern                                 | Description                                                                      |
| ------ | ------ | -------- | ---------- | ---------- | --------------------------------------- | -------------------------------------------------------------------------------- |
| `iban` | string | Yes      | 1          | 30         | `[A-Z]{2,2}[0-9]{2,2}[a-zA-Z0-9]{1,30}` | International Bank Account Number (IBAN) of the debtor account that was debited. |

#### Example SEPA Direct Debit Payment Settled webhook request body

```json
{
  "Type": "Sepa.DirectDebits.Payment.Settled",
  "Version": 1,
  "Payload": {
    "paymentId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "transactionId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "settlementDate": "2024-06-01",
    "debitCreditCode": "Debit",
    "interbankSettlementAmount": {
      "amount": 1224.48,
      "currency": "EUR"
    },
    "debtorAccount": {
      "iban": "GB24BKEN10000031510604"
    }
  },
  "Nonce": 188104452
}
```

#### Example webhook response

```json
{
  "Nonce": 188104452
}
```
