---
title: "SEPA DD Debit Payment Collection Pending webhook"
version: 1
webhook: true
---

Sent when an inbound SEPA Direct Debit collection is imminent.

#### Request body

```json
{
  "Type": "Sepa.DD.DebitPayment.CollectionPending",
  "Version": 1,
  "Payload": {...},
  "Nonce":
}
```

#### Payload Fields

| Field                          | Type   | Format | Required | Min length | Max length | Pattern                        | Description                                                                                                                                                                                              |
| ------------------------------ | ------ | ------ | -------- | ---------- | ---------- | ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `transactionId`                | string | uuid   | Yes      | -          | -          | -                              | Unique ClearBank identifier for the transaction that will be applied to the advised Debtor Account. System-generated; also returned in the Debit Payment Completed webhook.                                      |
| `paymentId`                    | string | uuid   | Yes      | -          | -          | -                              | Unique ClearBank identifier for the payment. System-generated; also returned in the Debit Payment Completed webhook.                                                                                             |
| `txId`                         | string | -      | Yes      | 1          | 35         | -                              | Unique identifier for the payment, as assigned by the scheme. In the SIM environment, this is a ClearBank-generated value.                                                                               |
| `endToEndId`                   | string | -      | Yes      | 1          | 35         | -                              | Non-unique identifier advised by the originating bank. Corresponds to the `endToEndId` submitted in the POST request.                                                                                    |
| `mandateId`                    | string | -      | Yes      | 1          | 35         | -                              | Unique identification, as assigned by the creditor, to unambiguously identify the mandate. Corresponds to the `mandateId` submitted in the POST request.                                                 |
| `creditorSchemeIdentification` | object | -      | Yes      | -          | -          | -                              | Unique and unambiguous identification of the creditor party who signed the mandate. See `CreditorSchemeIdentification`.                                                                                  |
| `instructedAmount`             | object | -      | Yes      | -          | -          | -                              | The instructed amount. See `InstructedAmount`.                                                                                                                                                           |
| `requestedCollectionDate`      | string | date   | Yes      | -          | -          | `YYYY-MM-DD`                   | Date at which the creditor requests that the amount is to be collected from the debtor.                                                                                                                  |
| `sequenceType`                 | string | enum   | Yes      | -          | -          | `FNAL`, `FRST`, `OOFF`, `RCUR` | Identifies the direct debit sequence. FNAL = Final, FRST = First, OOFF = One-off, RCUR = Recurrent.                                                                                                      |
| `debtor`                       | object | -      | Yes      | -          | -          | -                              | Party that owes an amount of money to the (ultimate) creditor. See `Party`.                                                                                                                              |
| `debtorAccount`                | object | -      | Yes      | -          | -          | -                              | Account of the debtor to which a debit entry will be made. See `PartyAccount`.                                                                                                                           |
| `creditor`                     | object | -      | Yes      | -          | -          | -                              | Party that is owed an amount of money. See `Party`.                                                                                                                                                      |
| `creditorAccount`              | object | -      | Yes      | -          | -          | -                              | Account of the creditor to which a credit entry will be made. See `PartyAccount`.                                                                                                                        |
| `creditorAgent`                | object | -      | Yes      | -          | -          | -                              | Financial institution servicing an account for the creditor. See `CreditorAgent`.                                                                                                                        |
| `transactionReference`         | string | -      | No       | 1          | 140        | -                              | The reference of the transaction that was credited to your account. Populated from the unstructured remittance information submitted with the payment. Absent if no remittance information was provided. |

##### CreditorSchemeIdentification

| Field            | Type   | Required | Min length | Max length | Description                                                                              |
| ---------------- | ------ | -------- | ---------- | ---------- | ---------------------------------------------------------------------------------------- |
| `identification` | string | Yes      | 1          | 35         | Creditor party that signs the mandate. Unique and unambiguous identification of a party. |

##### InstructedAmount

| Field      | Type    | Required | Constraints       | Pattern      | Description                                                           |
| ---------- | ------- | -------- | ----------------- | ------------ | --------------------------------------------------------------------- |
| `amount`   | decimal | Yes      | 0.01–999999999.99 | -            | The instructed amount. Must be 0.01 or more and 999999999.99 or less. |
| `currency` | string  | Yes      | Must be `EUR`     | `^[A-Z]{3}$` | Currency of the instructed amount. Only EUR is allowed.               |

##### Party (Debtor and Creditor)

| Field           | Type   | Required | Min length | Max length | Description                                                                                                       |
| --------------- | ------ | -------- | ---------- | ---------- | ----------------------------------------------------------------------------------------------------------------- |
| `name`          | string | Yes      | 1          | 70         | The name used to identify the legal owner of the account.                                                         |
| `postalAddress` | object | No       | -          | -          | Address of the party. Present only if a postal address was provided on the original payment. See `PostalAddress`. |

##### PostalAddress (within webhook Party)

All six fields are always present in the payload. Fields that were not provided on the original payment are serialised as empty strings (`""`).

| Field            | Type   | Required | Min length | Max length | Description                                                                                                                                                           |
| ---------------- | ------ | -------- | ---------- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `streetName`     | string | Yes      | 1          | 70         | Name of a street or thoroughfare. Always non-empty (required on the original request).                                                                                |
| `country`        | string | Yes      | 2          | 2          | Nation with its own government. ISO 3166-1 alpha-2 country code. Always non-empty (required on the original request).                                                 |
| `buildingNumber` | string | Yes*     | 1          | 16         | Number that identifies the position of a building on a street. Empty string if not provided on the original request.                                                  |
| `buildingName`   | string | Yes*     | 1          | 35         | Name of the building or house. Empty string if not provided on the original request.                                                                                  |
| `postCode`       | string | Yes*     | 1          | 16         | Identifier consisting of a group of letters and/or numbers added to a postal address to assist sorting of mail. Empty string if not provided on the original request. |
| `townName`       | string | Yes*     | 1          | 35         | Name of a built-up area, with defined boundaries, and a local government. Empty string if not provided on the original request.                                       |

\*- Always present in the payload, but value may be an empty string.

##### PartyAccount (DebtorAccount and CreditorAccount)

| Field  | Type   | Required | Min length | Max length | Pattern                                 | Description                                                                 |
| ------ | ------ | -------- | ---------- | ---------- | --------------------------------------- | --------------------------------------------------------------------------- |
| `iban` | string | Yes      | 1          | 30         | `[A-Z]{2,2}[0-9]{2,2}[a-zA-Z0-9]{1,30}` | International Bank Account Number (IBAN) of the debtor or creditor account. |

##### CreditorAgent

| Field | Type   | Required | Min length | Max length | Pattern                                                      | Description                                                                                                         |
| ----- | ------ | -------- | ---------- | ---------- | ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| `bic` | string | Yes      | 8          | 11         | `^[A-Z0-9]{4,4}[A-Z]{2,2}[A-Z0-9]{2,2}([A-Z0-9]{3,3}){0,1}$` | Business Identifier Code (BIC) of the financial institution servicing the creditor account, as defined by ISO 9362. |
