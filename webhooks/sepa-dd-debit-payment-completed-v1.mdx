---
title: "SEPA DD Debit Payment Completed webhook"
version: 1
webhook: true
---

Sent when a SEPA Direct Debit payment is collected from an account. Follows the SEPA DD Debit Payment Collection Pending webhook.

**Note:** You can correlate settled payments with the original collection using the `paymentId` and `transactionId` as these correspond to the same fields in the Payment Collection Pending webhook.

#### Request body

```json
{
  "Type": "Sepa.DD.DebitPayment.Completed",
  "Version": 1,
  "Payload": {...},
  "Nonce":
}
```

#### Payload Fields

| Field                       | Type   | Format | Required | Min length | Max length | Description                                                                                                                                                     |
| --------------------------- | ------ | ------ | -------- | ---------- | ---------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `PaymentId`                 | string | uuid   | Yes      | -          | -          | Unique ClearBank identifier for the transaction we will apply on to the advised Debtor Account.                                                                 |
| `TransactionId`             | string | uuid   | Yes      | -          | -          | Unique identification, assigned by ClearBank, to unambiguously identify the transaction.                                                                        |
| `SettlementDate`            | string | date   | Yes      | -          | -          | Date on which the amount of money ceases to be available to the agent that owes it and becomes available to the agent to which it is due. Format: `YYYY-MM-DD`. |
| `InterbankSettlementAmount` | object | -      | Yes      | -          | -          | Settled amount in the interbank space. See `InstructedAmount`.                                                                                                  |
| `DebtorAccount`             | object | -      | Yes      | -          | -          | Account of the debtor to which the debit entry was applied. See `PartyAccount`.                                                                                 |

##### InstructedAmount

| Field      | Type    | Required | Constraints       | Pattern      | Description                                                        |
| ---------- | ------- | -------- | ----------------- | ------------ | ------------------------------------------------------------------ |
| `Amount`   | decimal | Yes      | 0.01–999999999.99 | -            | The settled amount. Must be 0.01 or more and 999999999.99 or less. |
| `Currency` | string  | Yes      | Must be `EUR`     | `^[A-Z]{3}$` | Currency of the settled amount. Only EUR is allowed.               |

##### PartyAccount (DebtorAccount)

| Field  | Type   | Required | Min length | Max length | Pattern                                 | Description                                                                      |
| ------ | ------ | -------- | ---------- | ---------- | --------------------------------------- | -------------------------------------------------------------------------------- |
| `Iban` | string | Yes      | 1          | 30         | `[A-Z]{2,2}[0-9]{2,2}[a-zA-Z0-9]{1,30}` | International Bank Account Number (IBAN) of the debtor account that was debited. |

#### Example SEPA DD Debit Payment Completed webhook request body

```json
{
  "Type": "Sepa.DD.DebitPayment.Completed",
  "Version": 1,
  "Payload": {
    "PaymentId": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
    "TransactionId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "SettlementDate": "2024-06-01",
    "InterbankSettlementAmount": {
      "Amount": 1224.48,
      "Currency": "EUR"
    },
    "DebtorAccount": {
      "Iban": "GB24BKEN10000031510604"
    }
  },
  "Nonce": 188104452
}
```

#### Example webhook response

```json
{
  "Nonce": 188104452
}
```
